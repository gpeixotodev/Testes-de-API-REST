Descrição do projeto

- Projeto voltado à validação de APIs REST da aplicação Urban Grocers, com foco na elaboração e execução de casos de teste para endpoints de cadastro de produtos em kits e cálculo de entregas. Também foram identificados e documentados bugs encontrados durante os testes.

1. Objetivo do projeto

- Validar o funcionamento dos endpoints da API, verificando respostas para cenários positivos e negativos, além de identificar inconsistências no tratamento dos dados enviados nas requisições.

2. Resultado

- Foram executados testes em diferentes endpoints utilizando requisições HTTP, validando códigos de resposta, regras de negócio e parâmetros de entrada. Durante os testes, foram identificados e registrados bugs relacionados à validação de dados e ao comportamento esperado da API.

3. Ferramentas utilizadas
- Postman;
- APIs REST;
- Requisições HTTP (POST);
- JSON;
- Jira;
- Casos de Teste (Test Cases).
4. O que aprendi com esse projeto (habilidades/competências adquiridas)

- Testes manuais de APIs REST;
- Criação e execução de requisições HTTP;
- Validação de códigos de status e respostas em JSON;
- Testes positivos e negativos de endpoints;
- Identificação e documentação de bugs em APIs.
  
5. Existem melhorias a serem feitas?

- Como evolução do projeto, seria interessante automatizar os testes utilizando coleções do Postman com scripts em JavaScript, integrar a execução em pipelines de CI/CD e ampliar a cobertura dos testes para outros métodos HTTP e cenários de exceção.

- Metodologia

- Análise da documentação da API, elaboração dos casos de teste, envio das requisições via Postman, validação das respostas da API e registro dos defeitos encontrados no Jira.

Como executar o projeto

- Importar a coleção da API no Postman, configurar o ambiente de testes, executar as requisições previstas nos casos de teste, validar os códigos de resposta e o conteúdo retornado em JSON e registrar eventuais inconsistências identificadas durante a execução.
